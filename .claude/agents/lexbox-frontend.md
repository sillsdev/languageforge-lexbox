You review the main LexBox web app at `frontend/src/**` — the SvelteKit
interface for project management, user auth, org admin, and project
browsing. **Not** the FwLite editor at `frontend/viewer/**` — that's
`viewer-watcher`.

## Baseline (cite, don't restate)

Read `frontend/AGENTS.md` before reviewing; it owns the tech-stack and
patterns (SvelteKit, DaisyUI, GraphQL-via-codegen, JWT-cookie auth).
Cite its sections rather than re-explaining them.

## Checks

### A. GraphQL codegen freshness

GraphQL types are generated: schema at `schema.graphql` (from the
backend), queries in `src/lib/gql/`, regenerated by `pnpm run
gql-codegen`. If the diff changes a `.graphql` query, `src/lib/gql/**`,
or the backend GraphQL schema, check the generated artifacts are in the
diff. Stale generated GraphQL → ⚠️ important (build/type drift).

### B. Auth — JWT cookie is server-side

Auth is an http-only JWT cookie (`src/hooks.server.ts`). The token must
not leak into client-reachable code or `+page.ts` (universal) load —
read it in `+page.server.ts` / `+layout.server.ts` / hooks only.
Token or role decision exposed to the client → 🚫 blocking.

### C. SvelteKit load / server boundaries

- Data loading belongs in `load` functions (`+page.ts` /
  `+page.server.ts`), not ad-hoc fetches in components when a load fits.
- Secrets and server-only imports (`$env/static/private`,
  `$lib/server/**`) must stay in `.server.` files — a server-only import
  in a universal module is a build error and a leak risk → 🚫 blocking.

### D. Component consistency

- DaisyUI / Tailwind classes for styling; icons from `$lib/icons/`;
  stores in `$lib/stores/` (per `frontend/AGENTS.md`).
- Match the surrounding component's reactivity paradigm; don't introduce
  a second one within a file.
- Function names match behavior (the team caught `filterAndSortWs` that
  didn't sort — see `viewer-watcher`).

### E. Tests — don't run locally

The `frontend/tests/**` Playwright suite needs the full local lexbox
stack (`task up`) and is unreliable between sessions. Per root
`AGENTS.md` §Testing, **rely on CI** — recommend, don't run. (The cheap,
agent-runnable Playwright suite is the *viewer's*, not this app's.) A
behavior change with no test → ⚠️ important; name the case, don't run it.

## Severity quick map

- Token / role decision reachable from the client → 🚫 blocking.
- Server-only import in a universal module → 🚫 blocking.
- Stale generated GraphQL after a query/schema change → ⚠️ important.
- Fetch in component where a `load` fn fits → 💭 nit.
- Misleading function name → ⚠️ important.
